# APEXCash — Plan de implementación Postflop V1

## 1. Objetivo de Postflop V1

Postflop V1 será la continuación natural del sistema Preflop V1. El objetivo no es crear todavía un solver completo ni un motor postflop perfecto, sino construir un entrenador práctico de spots postflop que enseñe al usuario a tomar mejores decisiones en flop, turn y river usando dos capas:

1. **Explicación GTO simplificada**
2. **Ajuste práctico Pool NL2-NL10**

El sistema debe mantener la misma filosofía que ya quedó funcionando en Preflop V1:

* Una decisión por spot.
* Feedback inmediato.
* Panel derecho con resultado, frecuencia, EV relativo y explicación.
* Bloque inferior con ajuste Pool NL2-NL10.
* Sistema de leaks por módulo.
* Filtros por módulo.
* Flujo simple: responder → ocultar botones → mostrar resultado → siguiente spot.

---

## 2. Principio clave

Postflop V1 no debe intentar jugar una mano completa todavía.

No vamos a crear un árbol completo flop-turn-river con todas las ramas posibles.

La V1 será un sistema de **spots postflop aislados**, igual que preflop:

```text
Situación concreta
Board concreto
Rangos aproximados
Acción previa
Hero debe decidir
Feedback
Siguiente spot
```

Esto permite avanzar rápido sin romper la arquitectura actual.

---

## 4. Módulos de Postflop V1

La primera versión debe tener módulos concretos y controlados.

### Módulo 1 — CBet IP

Hero fue agresor preflop y juega en posición.

Ejemplos:

* BTN vs BB
* CO vs BB
* BTN vs SB

Tipos de boards:

* A-high secos
* K-high secos
* Boards bajos secos
* Boards conectados
* Boards monotono
* Boards emparejados

Objetivo del módulo:

Enseñar cuándo apostar pequeño, cuándo checkear back y cuándo apostar grande.

Opciones iniciales:

```php
['CHECK', 'BET_33', 'BET_75']
```

---

### Módulo 2 — CBet OOP

Hero fue agresor preflop pero juega fuera de posición.

Ejemplos:

* SB 3Bet vs BTN
* BB 3Bet vs BTN
* UTG open vs BTN call

Objetivo:

Enseñar que fuera de posición no se puede apostar automáticamente todos los boards.

Opciones:

```php
['CHECK', 'BET_33', 'BET_75']
```

---

### Módulo 3 — BB vs CBet

Hero defendió BB y enfrenta c-bet.

Ejemplos:

```text
BTN open
BB call
Flop
BB check
BTN bets
Action on Hero
```

Objetivo:

Enseñar cuándo foldear, pagar o check-raisear.

Opciones:

```php
['FOLD', 'CALL', 'RAISE']
```

---

### Módulo 4 — Turn Barrel

Hero apostó flop y llega el turn.

Objetivo:

Enseñar cuándo seguir apostando, cuándo rendirse y cuándo controlar bote.

Opciones:

```php
['CHECK', 'BET_50', 'BET_75']
```

---

### Módulo 5 — River Value Bet

Hero llega al river con mano de valor.

Objetivo:

Enseñar a apostar por valor contra el pool NL2-NL10, donde muchos jugadores pagan demasiado.

Opciones:

```php
['CHECK', 'BET_50', 'BET_75']
```

---

### Módulo 6 — River Bluff Catch

Hero enfrenta apuesta river.

Objetivo:

Enseñar cuándo pagar y cuándo foldear, especialmente porque en NL2-NL10 el pool farolea menos de lo que debería.

Opciones:

```php
['FOLD', 'CALL']
```

---

## 5. Módulos que NO entran todavía

No meter en Postflop V1:

* Donk bet
* Overbet avanzada
* Check-raise turn
* Multiway pots
* Side pots
* SPR complejo
* All-in trees
* Ranges dinámicos completos
* Solver real

Eso sería Postflop V2.

---

## 6. Primer módulo a implementar

El primer módulo debe ser:

```text
CBet IP
```

Razones:

* Es el más fácil de entender.
* Es el más frecuente.
* Conecta directamente con preflop.
* Permite usar boards simples.
* Da feedback claro.
* Tiene mucho valor para NL2-NL10.

## 8. Primeros spots recomendados

### Spot 1

```text
BTN abre
BB paga
Flop K72 rainbow
Hero tiene AKo
BB check
```

Decisión correcta:

```text
BET_33
```

Idea:

Top pair top kicker en board seco. Apostar pequeño imprime valor.

Pool NL2-NL10:

Muchos jugadores pagan demasiado con Kx peor, 7x, pocket pairs y floats débiles. Apostar pequeño por valor es mejor que slowplay.

---

### Spot 2

```text
BTN abre
BB paga
Flop K72 rainbow
Hero tiene AQo
BB check
```

Decisión correcta:

```text
BET_33 o CHECK según estructura simplificada
```

Idea:

Tiene overcards y ventaja de rango, pero no valor directo.

Pool NL2-NL10:

Contra jugadores que foldean mucho flop, c-bet pequeña funciona. Contra calling stations, check back es mejor porque no hay fold equity suficiente.

---

### Spot 3

```text
CO abre
BB paga
Flop A72 rainbow
Hero tiene TT
BB check
```

Decisión correcta:

```text
CHECK
```

Idea:

Tenemos showdown value, pero apostar convierte la mano en bluff sin necesidad.

Pool NL2-NL10:

Muchos rivales no foldean Ax ni pares. Check back controla bote y evita aislarse contra manos mejores.

---

### Spot 4

```text
BTN abre
BB paga
Flop T92 con flush draw
Hero tiene QJs con proyecto
BB check
```

Decisión correcta:

```text
BET_75
```

Idea:

Proyecto fuerte con mucha equity. Buen candidato para apostar.

Pool NL2-NL10:

Semi-bluff rentable si el rival foldea suficiente. Contra calling stations, sigue siendo buena apuesta porque tenemos equity real, no puro aire.

---

### Spot 5

```text
BTN abre
BB paga
Flop A83 rainbow
Hero tiene QJs sin backdoors claros
BB check
```

Decisión correcta:

```text
CHECK
```

Idea:

La mano tiene poca equity y mala continuación.

Pool NL2-NL10:

No hace falta farolear todos los boards A-high. Contra rivales que pagan demasiado, rendirse con aire malo ahorra dinero.

Agregar botón:

```html
<button type="button" data-module="cbet_ip">CBet IP</button>
```

Añadir:

```php
'cbet_ip' => 'CBet IP',
```

### 5. Verificar frontend

El frontend ya debería soportar:

* cartas de Hero
* acción previa
* opciones dinámicas
* resultado
* EV
* frecuencia
* Pool NL2-NL10

Pero falta mostrar el board real si aún no existe visualmente.

## 10. Cambio visual necesario

Actualmente la mesa muestra:

```text
SPOT PREFLOP
Pot: X BB
```

Para postflop debe mostrar board:

```text
FLOP
K♥ 7♣ 2♦
Pot: 6.5 BB
```

Se debe añadir un contenedor:

```html
<div class="board-cards" id="boardCards"></div>
```

Y en JS:

```js
if (spot.board && spot.board.length) {
    renderBoard(spot.board);
}
```

Si no hay board, se mantiene:

```text
SPOT PREFLOP
```

## 11. No romper Preflop

El sistema debe ser compatible con ambos tipos:

```php
'street' => 'preflop'
```

o:

```php
'street' => 'flop'
```

Si un spot no tiene `board`, el frontend debe comportarse igual que ahora.

No modificar los módulos preflop para adaptarlos a postflop.

## 15. Definición de terminado para Postflop V1 inicial

La primera entrega estará lista cuando:

* Exista módulo `CBet IP`.
* Se puedan cargar spots de flop.
* Se vea el board en la mesa.
* El usuario pueda responder.
* Se oculten botones al responder.
* Se muestre resultado en panel derecho.
* Se muestre Pool NL2-NL10 debajo de la mesa.
* `Siguiente spot` cargue otro spot.
* Los filtros funcionen.
* Preflop siga funcionando sin romperse.

---

## 16. Filosofía final

Postflop V1 debe enseñar al usuario a pensar en:

```text
¿Qué representa mi rango?
¿Qué representa el board?
¿Qué tipo de mano tengo?
¿Qué manos peores pagan?
¿Qué manos mejores foldean?
¿El pool NL2-NL10 realmente foldea aquí?
¿Estoy apostando por valor, bluff o protección?
```

No debe enseñar a memorizar botones.

Debe enseñar a tomar mejores decisiones.
