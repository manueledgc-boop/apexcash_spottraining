RECORDERIS - APEXCASH / HAND LAB V1
Fecha: 2026-06-21

ESTADO ACTUAL

Se cerró una fase importante del Hand Lab V1: integración de familias de spot para preflop.

Lo que quedó hecho:

1. SpotFamilyResolver centralizado
   - Archivo: app/SpotTraining/SpotFamilyResolver.php
   - Namespace usado: App\SpotTraining
   - Todos los módulos preflop deben consumir esta clase.
   - Ya no se deben repetir métodos spotFamilyFromConcept() ni spotFamilyLabelFromConcept() dentro de cada módulo.

2. Familias preflop V1 simplificadas y unificadas
   Familias activas:
   - premium_or_strong_value
   - suited_broadway
   - broadway_offsuit
   - ax_suited
   - ax_offsuit
   - medium_pair
   - small_pair
   - suited_connector
   - offsuit_connector
   - weak_suited_broadway
   - weak_suited_connector
   - blind_defense
   - trash_offsuit
   - uncategorized_preflop

   Decisión importante:
   No usar por ahora familias demasiado finas como strong_broadway_suited, medium_broadway_suited o premium_broadway.
   Para V1 es mejor consistencia y seguridad antes que exceso de precisión.

3. Biblioteca oficial preflop actualizada
   Todos los módulos preflop deben tener dentro del array del spot:

   'spot_family' => SpotFamilyResolver::fromConcept($concept),
   'spot_family_label' => SpotFamilyResolver::labelFromConcept($concept),

   En archivos que usan fromData(array $spot), debe usarse:

   'spot_family' => SpotFamilyResolver::fromConcept($spot['concept']),
   'spot_family_label' => SpotFamilyResolver::labelFromConcept($spot['concept']),

   No usar $concept si no existe en esa función.

4. Archivos preflop tocados/actualizados
   - BtnVs3BetSpots.php
   - ThreeBetVsOpenSpots.php
   - BbVsBtnSpots.php
   - SbVsBtnSpots.php
   - BbVsSbSpots.php
   - OpenRaiseSpots.php

5. Hand Lab empezó a guardar spot_family
   Se añadió/planificó columna en hand_lab_spots:
   - spot_family
   - spot_family_label

   En HandLabSpot.php deben estar en $fillable:
   - spot_family
   - spot_family_label

6. HandLabController
   Después de validar el request y antes de llamar al SimilarityService, debe calcular:

   if (strtolower($validated['street']) === 'preflop') {
       $validated['spot_family'] = SpotFamilyResolver::fromCards($validated['hero_cards']);
       $validated['spot_family_label'] = SpotFamilyResolver::labelFromFamily($validated['spot_family']);
   } else {
       $validated['spot_family'] = null;
       $validated['spot_family_label'] = null;
   }

   Esto debe estar antes de:
   $match = $this->similarity->findBestMatch($validated);

7. SimilarityService
   Objetivo actual:
   Para preflop, el match debe ser estricto.

   Regla definitiva:
   - street debe coincidir.
   - pot_type debe coincidir.
   - hero_position debe coincidir.
   - villain_position debe coincidir.
   - spot_family debe coincidir.

   Si street = preflop y falta spot_family en payload o candidato:
   -> return 0

   Si street = preflop y spot_family es distinta:
   -> return 0

   Esto evita errores como responder JTo offsuit con una explicación de KTs suited.
