# 4. Servicios Laravel obligatorios

## 4.1 CashSessionService

Responsable de:

```text
crear Cash Session
validar bankroll
descontar buy-in
finalizar sesión
devolver stack Hero al bankroll
cerrar sesión por bust
cerrar sesión por abandono
```

Métodos mínimos:

```php
startSession(User $user, string $difficulty, int $buyinUnits): CashSession
endSession(CashSession $session, string $reason = 'finished'): void
handleHeroBusted(CashSession $session): void
getActiveSession(User $user): ?CashSession
```

---

Responsable de:


# 5. Endpoints y errores

## 5.1 Formato estándar success

```json
{
  "success": true,
  "data": {}
}
```

---

## 5.2 Formato estándar error

```json
{
  "success": false,
  "code": "INSUFFICIENT_BANKROLL",
  "message": "Bankroll insuficiente para entrar a mesa."
}
```

---

## 5.3 Códigos de error

```text
INSUFFICIENT_BANKROLL
INVALID_BUYIN
NO_ACTIVE_SESSION
SESSION_ALREADY_ACTIVE
SESSION_NOT_FOUND
HAND_NOT_FOUND
HAND_ALREADY_FINISHED
NOT_HERO_TURN
INVALID_ACTION
INVALID_AMOUNT
PLAYER_NOT_FOUND
BOT_LOOP_GUARD
HERO_BUSTED
TABLE_NOT_READY
SERVER_ERROR
```

---

## 5.4 POST /cash/session/start

Entrada:

```json
{
  "difficulty": "normal",
  "buyin_units": 150
}
```

Validaciones:

```text
difficulty in normal, hard
buyin_units >= 100
buyin_units <= 200
bankroll >= buyin_units
no active cash session
```

Acciones:

```text
crear cash_session
descontar buy-in
crear cash_table_players
crear bots iniciales
```

Salida:

```json
{
  "success": true,
  "data": {
    "cash_session_id": 1,
    "redirect": "/cash/table"
  }
}
```

---

## 5.5 POST /cash/session/end

Entrada:

```json
{
  "reason": "finished"
}
```

Acciones:

```text
si hay mano terminada no guardada, guardar
devolver stack Hero al bankroll
marcar cash_session como finished/abandoned/hero_busted
limpiar estado activo
```

---

## 5.6 POST /cash/hand/start

Acciones:

```text
validar sesión activa
validar Hero stack > 0
validar mínimo 2 jugadores activos
reemplazar bots si corresponde
crear mano
devolver safeHand
```

---

## 5.7 POST /cash/action

Entrada:

```json
{
  "seat_id": 0,
  "action": "CALL",
  "amount_units": 4
}
```

Validaciones:

```text
sesión activa
mano activa
turno de Hero
acción válida
amount válido
```

Salida:

```json
{
  "success": true,
  "data": {
    "hand": {}
  }
}
```

El hand devuelto siempre debe ser safeHand.

---

## 5.8 POST /cash/bot-step

Acciones:

```text
ejecutar bots hasta Hero o hand_over
devolver safeHand
```

---

## 5.9 GET /cash/session/stats

Devuelve:

```json
{
  "success": true,
  "data": {
    "table": "Difícil",
    "hands": 120,
    "vpip": 24.5,
    "pfr": 20.1,
    "three_bet": 8.2,
```

## 5.10 GET /cash/session/last-hands

Devuelve últimas 10 manos formato panel.

# 6. Estados de UI

## 6.1 Sin bankroll

Mostrar depósito.

No permitir entrar a mesa.

---

## 6.2 Cash Session activa

Si usuario intenta ir a Dashboard, Historial o Estadísticas:

```text
Mostrar modal:
¿Desea finalizar la sesión actual?
[Cancelar]
[Finalizar sesión]
```

---

## 6.3 Hero busted

Mostrar:

```text
Has perdido tu stack de la mesa.
```

Si bankroll >= 50 BB:

```text
Volver al Lobby Cash
```

Si bankroll < 50 BB:

```text
Depositar 100 BB
```

---

## 6.4 Error API

Mostrar mensaje discreto.

No romper layout.

Ejemplo:

```text
No se pudo procesar la acción. Inténtalo de nuevo.
```

---

## 6.5 Pérdida de conexión

Mostrar:

```text
Reconectando...
```

Si no se recupera:

```text
Finalizar Cash Session como abandoned en próximo request válido.
```

# 7. Cierre por pestaña/navegador

Usar:

```js
navigator.sendBeacon('/cash/session/end')
```

También:

```js
beforeunload
visibilitychange
```

Fallback backend:

```text
Si existe cash_session active antigua al entrar nuevamente:
    cerrarla como abandoned
    devolver stack registrado de Hero si existe
```

---

# 8. Seguridad

Reglas obligatorias:

```text
Frontend nunca decide ganador.
Frontend nunca decide stack final.
Frontend nunca calcula rake final.
Frontend nunca recibe cartas ocultas.
Frontend nunca puede actuar por bots.
Backend valida turno.
Backend valida acción.
Backend valida amount.
Backend es única fuente de verdad.
```

# 12. Criterio final de terminado

No se considera terminado hasta que:

```text
1. Hero puede depositar.
2. Hero puede entrar con 50-100 BB.
3. Buy-in se descuenta.
4. Mesa tiene Hero + bots.
5. Bots tienen 50-250 BB iniciales.
6. Bots persisten entre manos.
7. Stacks persisten.
8. Hero puede jugar mano completa.
9. Showdown funciona.
10. Side pots funcionan.
11. Rake funciona.
12. No flop no drop funciona.
13. Bot busted se reemplaza tras 1 mano.
14. Hero busted redirige correctamente.
15. Salir devuelve stack al bankroll.
16. Mano se guarda completa.
17. Stats salen de datos reales.
18. Panel muestra últimas 10 manos.
19. Frontend no recibe cartas ocultas.
20. Historial completo muestra detalle.
21. Estadísticas completas funcionan.
22. 100 manos seguidas se juegan sin romperse.
```
