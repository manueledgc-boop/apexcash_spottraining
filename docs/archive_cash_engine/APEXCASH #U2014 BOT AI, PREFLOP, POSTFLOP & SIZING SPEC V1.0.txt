# APEXCASH — BOT AI, PREFLOP, POSTFLOP & SIZING SPEC V1.0

## Documento técnico de comportamiento de bots

---

# 1. Principio general

Los bots no deben jugar aleatoriamente.

Deben simular perfiles realistas de cash 6-Max.

La base estratégica se inspira en fundamentos estándar de escuelas modernas de cash games.

No se busca GTO perfecto en V1.

Se busca población realista:

```text
Recreacionales cometen errores
Nits foldean demasiado
Calling stations pagan demasiado
TAG juega sólido
Reg sólido juega bastante bien
```

# 3. Perfiles de bots

## Recreacional Pasivo

Características:

```text
VPIP alto
PFR bajo
Mucho call
Poca agresión
Pocas 3bets
Paga proyectos mal
Bluff bajo
```

Objetivo aproximado:

```text
VPIP 35-45
PFR 8-15
3Bet 2-4
```

---

## Recreacional Agresivo

```text
VPIP alto
PFR medio-alto
3bet errática
bluffs malos
overplays
```

Objetivo:

```text
VPIP 35-50
PFR 20-30
3Bet 6-10
```

---

## Calling Station

```text
VPIP alto
PFR bajo
Fold bajo
Call excesivo
Casi no bluffea
```

Objetivo:

```text
VPIP 40-55
PFR 5-12
3Bet 1-3
Fold vs CBet bajo
```

---

## Nit

```text
VPIP bajo
PFR bajo
3Bet value-heavy
Foldea demasiado
Pocos bluffs
```

Objetivo:

```text
VPIP 14-18
PFR 11-15
3Bet 3-5
```

---

## TAG

```text
Juego sólido
Rangos correctos
Agresión razonable
Pocos errores graves
```

Objetivo:

```text
VPIP 22-26
PFR 18-22
```


# 5. Ajustes por perfil preflop

## Nit

```text
Reducir open ranges 20-30%
3bet solo valor
Casi no call OOP
Fold BB alto
```

## Calling Station

```text
Más limp/call
Menos open raise
Menos 3bet
Call demasiado vs open
Defiende demasiadas manos suited
```

## Recreacional Pasivo

```text
Más limp
Más call
Pocas raises
Fold bajo en BB
```

## Recreacional Agresivo

```text
Open demasiado amplio
3bet más alto pero desequilibrado
4bet bluff ocasional absurdo
Overplay broadways y Ax
```

## TAG

```text
Usa rangos base
Ajustes mínimos
```

## Regular Sólido

```text
Usa rangos base
Mejor defensa BB
Más 3bet selectivo
Mejor selección de blockers
```

# 6. Facing open

## TAG / Reg sólido

Contra open:

```text
3bet por valor
3bet con blockers
call con manos jugables IP
fold basura
```

Preferencias:

```text
IP puede pagar más
OOP 3bet o fold más frecuente
SB casi raise/fold
BB defiende más amplio
```

# 9. Blind defense

## BB vs BTN

TAG / Reg sólido defiende amplio.

Defensa aproximada:

```text
Call: suited connectors, suited gappers, broadways, pares bajos/medios, Ax suited
3Bet: valor + blockers
Fold: offsuit basura
```

## SB

SB juega más raise/fold.

Pocos calls.

# 10. Sizing preflop

## Open raise

```text
UTG/HJ/CO/BTN = 2.5 BB
SB = 3 BB
```

Recreacional puede usar:

```text
2 BB
3 BB
4 BB
limp
```

## 3Bet

```text
IP = 3x open
OOP = 4x open
```

## Squeeze

```text
4.5x open + 1x por caller
```

## 4Bet

```text
IP = 2.2x 3bet
OOP = 2.4x 3bet
```

## All-in

Permitido si:

```text
SPR bajo
stack efectivo corto
raise/call deja comprometido
```

# 15. River strategy

River se divide en:

```text
value
bluff
check
bluffcatch
fold
```

## Value

Apostar:

```text
top pair buen kicker en spots favorables
overpair
dobles+
sets+
straights+
flushes+
```

## Bluff

Solo con:

```text
blockers
missed draws adecuados
rango percibido fuerte
línea coherente
```

## Calling Station

```text
paga demasiado river
bluffea poco
```

## Nit

```text
foldea demasiado
apuesta muy value-heavy
```

## Reg sólido

```text
elige bluffs con blockers
foldea bluffcatchers malos
value thin razonable
```

# 16. Sizing postflop

## Flop

```text
25-33% boards secos
33-50% boards medios
50-75% boards húmedos
66-100% boards muy húmedos
```

## Turn

```text
50-66% estándar
75% value/protection
100-125% polarizado
```

## River

```text
50-66% thin value
75-100% value fuerte
125-150% polarizado/overbet
All-in con SPR bajo
```

---

# 17. All-in postflop

Permitido si:

```text
SPR <= 1.5
mano fuerte
draw fuerte con fold equity
river value polarizado
```

Recreacionales pueden hacerlo mal.

Regs deben hacerlo con lógica.

# 20. Regla final

El preflop de los regs debe ser estricto.

Si los regs tienen mal preflop, ApexCash entrenará mal al usuario.

El objetivo no es que todos los bots jueguen perfecto.

El objetivo es que cada perfil juegue de forma coherente con lo que representa.
